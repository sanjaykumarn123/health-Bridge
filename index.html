Out-of-network handling: When a requesting provider is not within the same federated network, the router applies elevated consent verification — requires explicit patient-signed FHIR consent, restricts data scope to encounter + condition + procedure only, enforces mandatory audit log entry, and notifies the patient's proxy if configured. Break-glass out-of-network is permitted only with dual-clinician sign-off and is flagged for compliance review within 24 hours.
Payer flow — insurance claim verification & prior authorisation
Payer API Gateway — BCBS & Insurers SEPARATE ENTRY POINT
Payers connect via dedicated Payer API Gateway at the Central Router — not the hospital edge. Org-level JWT auth, standing consent, scoped access, 60-day auto-expiry. Handles both in-network (contracted providers) and out-of-network (non-contracted) claims with differentiated consent requirements.

Out-of-network payer claims: Non-contracted insurers must supply a patient-signed FHIR consent document at request time. Data scope is further restricted — only Encounter and Condition resources are returned. Mental health, substance use, and HIV-related records are always excluded regardless of consent. All OON payer requests are rate-limited to 10/hour per org and flagged for compliance review.
